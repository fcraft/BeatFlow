# 虚拟人体模块功能拓展计划

> 创建日期：2026-04-11
> 状态标记：✅ 已完成 | 🚧 进行中 | 📋 待开发

---

## 一、快捷操作区功能增强

### ✅ 1. 临床场景预设（Scenario Presets）
一键触发多命令组合的教学场景：
- 急性心梗（缺血 + 心动过速 + PVC）
- 心衰加重（心力衰竭 + 房颤 + 脱水）
- 运动诱发室速（高强度运动 → VT）
- 高钾血症危象（高钾 + 心动过缓 + T 波高尖）
- 过敏性休克（心动过速 + 低血压 + 支气管痉挛）
- 健康基线（一键重置所有状态）

**实现方式**：纯前端组合已有后端命令，无需后端改动。

### ✅ 2. 操作区 UX 改造
- 情绪行增加当前状态高亮（从 vitals 派生）
- 病变行增加当前节律状态高亮
- 紧急行增加「电复律」按钮
- 新增「快捷用药」行（β-blocker / 阿托品）
- 新增「体内状态」行（咖啡因 / 发热 / 脱水）
- 场景预设卡片区占用下方空白

## 二、中期功能建议

### 📋 3. 操作历史时间线（Action Timeline）
- 记录每次命令发送的时间和类型
- 可视化展示操作序列与 vitals 变化关联
- 支持导出为教学报告

### 📋 4. 教学引导模式（Guided Mode）
- 根据当前 vitals 自动推荐下一步操作
- 匹配 ACLS 流程（VF → 除颤 → 药物 → 再除颤）
- 智能提示卡片显示在快捷操作区

### 📋 5. 生理状态智能摘要（Smart Summary）
- 自动诊断文本：根据 PR/QRS/QT + 节律 → "窦性心动过速，HR 121，QTc 正常"
- 趋势箭头指示 HR↑ BP↓
- 替代右下角简单 beat info

---

## 三、远期功能建议

### 📋 6. 快捷截图/标注（Snapshot & Annotate）
- 一键冻结 ECG/PCG 波形截图
- 支持标记 P/QRS/T 波位置
- 导出为图片用于教学/论文

### 📋 7. 前后端命令一致性修复
以下命令前端 UI 中有控件但后端 `apply_command` 中无对应处理器：
- `condition_ischemia`（心肌缺血）
- `condition_pulm_hypertension`（肺高压）
- `condition_rv_failure`（右心衰）
- `set_fio2`（吸入氧浓度）
- `set_magnesium`（血清镁）
- `set_fitness`（体能水平）
- `set_age`（年龄）

需后端补齐实现。
